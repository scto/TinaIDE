# 依赖包管理说明

本文基于当前包管理页源码整理，重点说明已安装包列表、Linux / Android 双平台状态、检查更新、详情页，以及批量卸载这些和当前 UI 对得上的能力。

## 先理解当前依赖包入口

当前 **设置 → 依赖包** 会进入独立的包管理页面，而不是普通设置卡片。

它更偏向：

- 管理已经安装过的包
- 查看每个包在 Linux / Android 两侧的状态
- 做更新、卸载和详情排查

如果你是想浏览全部可安装包，当前空列表提示会引导你去首页 **市场**。所以不要把这里理解成完整包市场。

## 当前首页能做什么

顶部栏当前提供：

- 搜索
- 检查更新
- 刷新
- 选择模式
- 有更新时的 **全部更新**

如果检测到可更新包，顶部会显示数量标记，并出现 **全部更新** 按钮。

## 列表页看什么

当前列表来源是**已安装包数据**，卡片上通常会看到：

- 包名
- 版本
- 简短描述
- 内置标记
- 包版本 / 上游库版本（有元数据时）
- Linux 平台状态
- Android 平台状态

### Linux 和 Android 两行是什么意思

当前卡片会把同一个包拆成两种平台状态：

- `Linux`：对应 PRoot / Linux 环境侧
- `Android`：对应 NDK / Android 侧

所以你可能会遇到同一个包：

- Linux 已安装，但 Android 未安装
- Android 已安装，但 Linux 还没装
- 两边都有更新

这不是重复显示，而是平台状态本来就独立。

### 按钮状态

每个平台当前会显示以下几类状态之一：

- **安装**
- **已安装**
- **安装中**
- **更新**

如果你只是想补齐某个平台的库，不一定要重装整个包。

## 详情页能看什么

点开某个包后，当前详情页会展示：

- 基本信息
  - 分类
  - 主页
- 平台信息
  - 安装类型
  - Linux 包名（apt / apk 等适用时）
  - 大小
  - ABI
- 版本信息
  - 包版本
  - 包修订
  - 上游项目
  - 库版本
  - 上游标签
- 依赖
- 更新日志

### 依赖是可点的

如果包声明了依赖，详情页里的依赖项当前是可点击跳转的。

这适合顺着依赖链继续排查还缺什么。

## 安装、更新、卸载时会发生什么

当前进度对话框会分阶段显示：

- Preparing
- Downloading
- Verifying
- Extracting
- Installing
- Completed / Failed

如果安装或更新成功，当前成功提示还会提醒你：

- **重新打开文件**

这是因为新装的头文件补全和编译链接效果，往往要在文件重新打开后更容易刷新出来。

## 检查更新和全部更新

当你点 **检查更新** 后，页面会收集当前可更新包。

如果有结果：

- 顶部显示更新数量
- 可以直接点 **全部更新**

全部更新会逐个执行，不是并发一次性全拉。

## 选择模式和批量卸载

长按包卡片，或者点顶部 **选择**，可以进入选择模式。

当前选择模式支持：

- 多选
- 全选 / 取消全选
- 按平台批量卸载
  - 批量卸载 Linux
  - 批量卸载 Android

注意：

- 当前公开出来的是**批量卸载**，不是批量安装

## 什么时候会看到“暂无已安装的包”

这表示当前包管理页没有读到任何已安装包。

按当前文案，下一步通常不是在这里反复刷新，而是：

1. 去首页 **市场**
2. 先安装你需要的包
3. 再回到包管理页做更新、卸载或状态查看

## 常见问题

### 为什么安装了包，补全或编译还是没变化

先检查：

1. 目标平台是不是装对了，Linux 和 Android 不要看错
2. 安装是否真的成功完成
3. 重新打开过相关文件没有
4. 构建日志里是否还有其他缺失依赖

### 为什么卸载时会警告还有别的包依赖它

因为当前卸载确认会先查询依赖它的其他包，并把结果列出来，避免你把依赖链直接拆坏。

### 为什么这里搜不到某个新包

因为当前包管理页主要管理 **已安装包**，不是完整包市场。

要找新包，先去首页 **市场**。

## Android 依赖包的 include 和源码使用

Android 依赖包按产物类型使用，不按 ABI 拆成多个逻辑包。

- `header` 包：安装后会暴露包内的 `include/` 目录。重新打开相关 C/C++ 文件后，clangd 补全和单文件编译会自动加入这些头文件搜索路径。
- `source` 包：安装后同样会暴露 `include/`，但包内 `src/` 不会被 MobileIDE 自动编进你的项目。你需要在 CMake、Makefile 或项目源码配置里显式加入对应源码文件。
- `static`、`shared`、`executable` 包：安装后会暴露 `lib/` 或可执行文件目录，并根据当前设备 ABI 选择可用内容；不匹配当前设备 ABI 的包会在安装前被拦截。

例如，安装 `nlohmann-json`、`glm`、`stb` 这类 `header` 包后，可以直接在代码中 include 对应头文件。
安装 `tinyxml2`、`fmt`、`imgui` 这类 `source` 包后，还需要把包内源码加入构建。

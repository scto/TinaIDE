# Git 基础

MobileIDE 当前的 Git 能力已经覆盖了日常项目开发的主流程，但它的使用方式更接近“移动端工作台 + 设置页 + 终端配合”，不是完整照搬桌面 Git 客户端界面。

## 你现在可以怎么用 Git

基于当前源码，Git 相关能力主要分布在这些位置：

- **项目页** - 支持 Git 导入
- **编辑工作台的 Git 面板** - 查看更改、暂存、提交、日志
- **远程仓库对话框 / 同步对话框** - 管理 remotes，执行 fetch / pull / push 等远程操作
- **设置 → Git 设置** - 管理 HTTPS 凭据与 SSH 密钥
- **独立终端** - 执行高级 Git 命令

## 最常见的工作流

### 1. 导入仓库

如果你要把远程项目带到手机上，优先用项目页的 **Git 导入**。

### 2. 修改代码

在编辑器中改动文件后，回到 Git 面板查看：

- 已修改文件
- 未跟踪文件
- 已暂存文件

### 3. 暂存并提交

当前 Git 面板支持：

- 暂存单个文件
- 取消暂存
- 暂存全部 / 取消暂存全部
- 输入提交信息后直接提交

### 4. 远程同步

当前实现已经支持常见远程操作，例如：

- `fetch`
- `pull`
- `push`

以及远程仓库管理相关能力。

## HTTPS 与 SSH

### HTTPS

适合：

- GitHub / GitLab / Gitee 等常见 HTTPS 仓库
- 使用 Token / 用户名 / 密码类认证方式

设置入口：

**设置 → Git 设置 → HTTPS**

### SSH

适合：

- 想长期稳定管理多个主机
- 使用 Ed25519 等 SSH 密钥
- 需要更接近桌面开发的认证体验

设置入口：

**设置 → Git 设置 → SSH**

当前实现已支持 SSH 密钥管理与主机绑定。

## Git 面板能帮你做什么

在当前工作台里，Git 面板更适合做这些事情：

- 看哪些文件变了
- 把变更分批暂存
- 录入 commit message
- 提交当前变更
- 打开远程同步相关能力
- 查看日志与分支信息

如果你只是做普通改动提交，这套流程通常已经够用。

## 什么时候应该切到终端

这些情况更适合直接进终端：

- 复杂 rebase / cherry-pick
- 子模块处理
- 高级 stash 操作
- LFS 或自定义 hook 相关问题
- 你已经有明确的 Git 命令工作流

## 常见问题

### Q: push / pull 失败怎么办？

优先检查：

1. 当前仓库远程地址是否正确
2. HTTPS 凭据或 SSH 密钥是否已配置
3. 网络是否可用
4. 远程分支权限是否正常

### Q: HTTPS 还是 SSH 更适合我？

- 偶尔同步、快速上手：先用 **HTTPS**
- 长期开发、多个远程主机：更推荐 **SSH**

### Q: 界面里做不了的 Git 操作怎么办？

直接打开独立终端执行命令即可。MobileIDE 的设计本身就允许图形化工作流和命令行工作流并存。

## 推荐的使用顺序

1. 先把 HTTPS 或 SSH 配好
2. 用 Git 面板完成日常状态查看、暂存和提交
3. 再在需要时使用远程同步
4. 最后把复杂场景交给终端

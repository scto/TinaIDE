# 设置总览

本文基于当前设置页源码整理，帮助你快速找到常用选项，也说明遇到问题时应该先去哪个分区排查。

## 设置首页现在长什么样

当前设置首页分为三组：

- **开发相关** - 编辑器、语言服务器、编译器、项目、存储、Git、插件、依赖包
- **使用体验** - 终端、AI、外观、快捷键
- **其他** - 帮助中心、开发者选项、关于

如果你只是想知道“某个功能到底在哪改”，先从这个分组记忆就够用了。

## 编辑器

适合调整代码编辑体验：

- 字体大小、自定义字体、编辑器主题
- 行号、自动换行、彩虹括号、最大渲染行数
- 代码折叠、空白字符显示
- 自动缩进、Tab 宽度、使用空格代替 Tab
- 编辑器硬件加速

如果你觉得“显示不顺眼”或“输入手感不对”，先看这里。

## 语言服务器

适合排查补全、跳转、诊断相关问题：

- clangd 运行模式
- background index、clang-tidy、头文件插入策略
- 补全样式、函数参数占位符
- 补全数量限制、大小写敏感
- 签名帮助、Inlay Hints、Semantic Tokens、折叠范围
- 远程 LSP 开关、主机、端口、工作区根路径
- 同步模式、同步方法、rsync 模块与端口、连接测试

如果你遇到“补全不准”“跳转失效”“大项目分析太慢”，优先看这里。

对应的专门设置说明可看：

- [LSP 设置说明](lsp-settings.md)

## 编译器

适合调整构建与工具链行为：

- 优化等级、编译线程数
- CMake / Make / clang-format 的运行模式
- CMake 构建类型、生成器、并行任务数
- 格式化风格
- 当前工具链与 sysroot 状态
- 导入工具链、导入 sysroot
- 重新部署开发环境

如果是“构建链路不正常”“工具链缺失”“格式化行为不对”，先看这里。

## 项目

适合管理项目级行为和原生工程元数据：

- 自动保存间隔
- 自动备份
- 新建项目默认源码位置
- 当前项目 APK 导出类型与重新识别
- Native 依赖路径
  - include 目录
  - library 目录
  - runtime 目录
- Native 构建参数
  - CFLAGS
  - CXXFLAGS
  - LDFLAGS
  - LDLIBS
  - CMake 参数

如果项目能打开但“构建参数总是不对”，这里很关键。

## 存储

适合管理 Linux 环境、导入状态和清理能力：

- 导入 rootfs 预设
- 从本地导入 Linux 系统
- 从 URL 导入 Linux 系统
- 当前 Linux 系统与配置档案
- 导入状态查看
- Linux rootfs 路径
- MT 访问开关
- 存储清理入口

如果你遇到“开发环境装不上”“空间被占满”“系统导入异常”，先看这里。

对应的专门说明可看：

- [Linux 系统与存储](linux-storage.md)

## Git

当前 Git 设置页分成两大块：

- **HTTPS**
  - 添加和管理凭据
- **SSH**
  - 生成密钥
  - 导入密钥
  - 默认密钥
  - 密钥列表
  - 主机绑定

如果你遇到 `push`、`pull`、私有仓库认证问题，先检查这里，再去终端里跑一次 `git status` 或 `git remote -v`。

## 插件与依赖包

这两个入口分别解决不同问题：

### 插件

适合管理扩展能力：

- 已安装插件列表
- 启用与禁用
- 从文件安装 `.mobileplug`
- 脚本 / 混合插件权限确认
- 插件日志
- 主题型插件
- LSP 插件依赖安装
- 插件卸载

当前设置页里的插件入口更偏向 **已安装插件管理**。

如果你是想浏览新的插件，通常先去首页 **市场**；不要把当前设置页理解成完整插件市场。

### 依赖包

适合查看和管理已经装过的依赖包，以及它们在 Linux / Android 两侧的状态：

- 搜索已安装包
- 检查更新与全部更新
- Linux / Android 双平台安装状态
- 包详情、依赖、更新日志
- 按平台批量卸载

如果当前列表是空的，页面会提示你先去首页 **市场** 浏览和安装新包。

如果你不知道该进“插件”还是“依赖包”，可以这样区分：

- 改编辑器主题、代码片段、扩展功能：去 **插件**
- 管开发环境里的工具和安装状态：去 **依赖包**

## 终端

适合调整命令行环境：

- 终端主题
- 字体与字号
- 后端模式
- Shell 类型
- Linux 工具健康状态
- 系统名称、包管理器、开发包状态
- Zsh 安装状态
- Locale 支持
- 交互语言
- 光标闪烁与速率

如果你遇到“终端打不开”“中文乱码”“Shell 不对”“Linux 工具不全”，先看这里。

对应的专门设置说明可看：

- [终端设置说明](terminal-settings.md)

## AI

适合管理模型接入和工具行为：

- AI 接入模式
- 渠道管理
- 当前模型
- 工具开关
- 自动放行危险工具
- 深度思考
- 预算 token、最大 token、温度
- 超时、重试次数、重试延迟
- 图片细节等级
- 系统提示词、总结提示词
- 工具管理

如果 AI 能聊天但“模型不对”“渠道失效”“工具调用异常”，优先检查这里。

## 外观

这里目前主要负责：

- 应用主题
- 调试工具栏位置

如果你只是想切浅色 / 深色 / 跟随系统，直接看这里。

## 快捷键

当前快捷键系统主要面向 **硬件键盘**，支持：

- 保存 / 保存全部
- 标签切换与关闭
- 撤销 / 重做
- 书签切换与跳转
- 恢复默认快捷键

如果你用的是软键盘，很多 `Ctrl` / `Shift` / `F2` 组合并不稳定，这是正常现象。

## 帮助中心

帮助中心本身支持：

- 分类浏览
- 搜索文档
- 直接阅读 Markdown 帮助文档

如果你不确定某个入口属于哪个分区，可以先在帮助中心搜关键词，例如：

- 设置
- 终端
- Git
- AI
- 日志

## 开发者选项

开发者选项默认不是总会显示。

当前逻辑是：

- 服务端允许时，才能开启
- 一般需要在 **关于** 页对版本信息在 **3 秒内连续点击 5 次** 后启用

进入后可以看到诊断、后端调试和测试工具相关选项。普通用户不建议随意修改。

对应的专门说明可看：

- [开发者选项说明](developer-options.md)

## 关于

这里主要放“版本、社区、日志、许可证”相关能力：

- 版本信息
- GitHub
- 导出日志
- 上传日志
- 查看 PRoot 日志
- 崩溃日志自动上传
- 清空日志
- 开源许可证

如果你要反馈问题，最常用的就是这里的日志能力。

对应的专门说明可看：

- [关于与日志](about-and-logs.md)

## 遇到问题时先去哪里

可以按这个顺序找：

- **补全 / 跳转 / 诊断问题** → 语言服务器
- **构建 / 工具链 / 格式化问题** → 编译器、项目
- **开发环境 / 空间 / rootfs 问题** → 存储
- **终端 / Shell / Linux 工具问题** → 终端
- **Git 认证问题** → Git
- **AI 渠道 / 模型 / 工具问题** → AI
- **日志导出 / 反馈 / 许可证** → 关于

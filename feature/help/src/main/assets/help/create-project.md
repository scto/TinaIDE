# 创建项目

本文说明当前 MobileIDE 中真实可用的项目创建与导入方式。

## 入口

进入 **项目** 页后，你通常会从这里开始：

- 点击右下角 **+** 按钮新建项目
- 选择 **导入本地目录**
- 选择 **Git 导入**

## 新建项目向导

当前向导是两步式流程：

1. **选择模板**
2. **填写配置**

在配置步骤中，常见可选项包括：

- 项目名称
- C++ 标准版本
- NDK API Level（仅 NDK 模板）

## 当前可用模板

### C++ 单文件

适合：

- 学习语法
- 快速验证代码片段
- 编写一个独立的控制台程序

特点：

- 结构最简单
- 不依赖 CMake
- 适合直接编译运行

### CMake 可执行程序

适合：

- 常规 C/C++ 工程
- 多源文件项目
- 需要 compile_commands.json 和 clangd 补全的场景

特点：

- 标准 CMake 结构
- 更适合长期维护
- 首次构建后通常会生成编译数据库供 LSP 使用

### CMake 库

适合：

- 抽离公共模块
- 练习库工程结构
- 编写示例程序 + 库代码的组合项目

特点：

- 更利于拆分头文件与实现文件
- 适合后续扩展为更复杂的模块化工程

### Make 可执行程序

适合：

- 已习惯 Makefile 工作流
- 想手动控制编译规则
- 需要与已有 Make 项目保持一致

特点：

- 更自由
- 但需要你自行维护 Makefile 逻辑

### NDK 共享库

适合：

- Android 原生库开发
- 需要指定 Native API Level 的项目
- 研究 JNI / NDK 构建链路

特点：

- 向导会提供 NDK 相关配置项
- 更适合原生库和 Android 集成场景

## 如何选择模板

如果你只是想尽快开始：

- 练习或小实验：选 **C++ 单文件**
- 正常 C/C++ 工程：选 **CMake 可执行程序**
- 想写公共模块：选 **CMake 库**
- 已有 Make 习惯：选 **Make 可执行程序**
- 做 Android Native：选 **NDK 共享库**

## 创建后会发生什么

项目创建完成后，应用会自动帮你完成这些事情：

- 解压模板内容到目标目录
- 写入项目元数据
- 记录构建系统类型
- 记录 C++ 标准等基础配置

对于 CMake 项目，首次构建后通常还会生成 compile_commands.json，供代码补全和跳转能力使用。

## 导入已有项目

### 导入本地目录

如果你已经有现成的工程目录，可以直接从项目页导入。

适合：

- 从电脑复制到手机的工程
- 外部存储中的已有项目
- 历史项目重新接管

### Git 导入

项目页当前也支持从 Git 仓库导入。

适合：

- 克隆自己的远程仓库
- 拉取示例项目
- 在手机上快速查看或修改已有仓库

如果远程仓库需要认证，请先在 **设置 → Git 设置** 中配置 HTTPS 凭据或 SSH 相关能力。

## 创建后的推荐动作

创建或导入项目后，建议立刻做这几件事：

1. 打开主源码文件确认编码与目录结构正常
2. 尝试执行一次 **构建**
3. 查看是否生成或更新了 compile_commands.json
4. 进入 **设置 → 语言服务器** 检查 clangd / 远程 LSP 配置
5. 如需命令行操作，打开独立终端进入项目根目录

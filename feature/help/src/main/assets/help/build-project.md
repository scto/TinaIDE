# 编译项目

本文描述的是 MobileIDE 当前支持的主流构建工作流：单文件、CMake、Make，以及 NDK 原生库场景。

## 当前支持的项目类型

- **单文件项目** - 适合快速编译一个 main.cpp
- **CMake 项目** - 推荐的大多数 C/C++ 工程形式
- **Make 项目** - 适合已有 Makefile 的工程
- **NDK 共享库项目** - 适合原生库开发

## 标准工作流

最常见的流程如下：

1. 保存当前修改
2. 点击顶部的 **运行**、**构建** 或 **调试**
3. 等待构建完成
4. 在底部 **构建日志** 查看完整输出
5. 在 **诊断** 面板查看 clangd 错误与警告

## CMake 项目

对于 CMake 项目，首次构建前后通常会发生这些事情：

- 进行配置
- 生成或更新构建目录
- 生成或更新 compile_commands.json
- 执行编译与链接

这也是为什么很多补全、跳转和诊断能力会在首次构建后更稳定。

## 单文件项目

单文件项目更适合：

- 语法练习
- 快速跑一个小程序
- 临时验证算法

这类项目结构最简单，但随着文件变多，建议迁移到 CMake 或 Make。

## Make 项目

如果项目已有自己的 Makefile，MobileIDE 也可以配合它工作。

适合：

- 保持现有工程结构不变
- 继续使用你已有的 Make 规则
- 在终端里执行更复杂的自定义目标

## 运行配置

当前工作台支持运行配置相关能力，常见配置项包括：

- 运行目标
- 命令行参数
- 环境变量
- 工作目录

如果你的程序需要特定输入文件、参数或环境变量，请先检查运行配置，而不是只盯着编译本身。

## 调试

点击 **调试** 后，会进入 LLDB 相关调试流程。

适合：

- 下断点
- 单步执行
- 查看调用栈
- 查看变量

如果调试器不可用，通常说明开发环境还没部署完整，或者当前设备的 PRoot / LLDB 环境需要重新准备。

## 终端中的自定义构建

下列场景更适合使用独立终端：

- 手动执行 cmake
- 运行 make test、make clean
- 自定义脚本构建
- 执行项目自带的工具链脚本

## 结果应该看哪里

### 构建日志

优先看这里：

- 编译器输出
- 链接报错
- 目标文件或可执行文件生成过程

### 诊断

适合看：

- 文件内具体错误
- clangd 语义告警
- 点击跳转到出错位置

## 常见问题

### 编译失败但编辑器没报错

可能是：

- 构建系统配置错误
- 头文件路径错误
- 链接阶段才暴露的问题

先看 **构建日志**，不要只看编辑区颜色提示。

### 有补全，但运行失败

说明语言服务工作正常，但：

- 运行配置不对
- 目标程序没成功链接
- 工作目录或参数不匹配

### 调试按钮不可用或报错

先检查：

- 开发环境是否已部署完成
- 设备剩余空间是否充足
- 是否需要在设置里重新部署开发环境

### 装了依赖包，但头文件补全或编译结果还是没变化

先检查：

1. Linux / Android 平台是不是装对了
2. 安装流程是否真的成功完成
3. 相关文件有没有重新打开
4. 构建日志里是否还有别的缺失依赖

当前包管理页安装成功后，本身也会提示你重新打开文件。

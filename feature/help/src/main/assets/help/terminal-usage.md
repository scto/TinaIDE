# 终端使用

当前 MobileIDE 的终端是 **独立页面**，支持多会话、项目目录启动和不同后端切换，不再适合按“底部固定终端标签”来理解。

## 终端适合做什么

你可以在终端里执行这些高频任务：

- 手动运行 `cmake` / `make`
- 执行 Git 命令
- 运行脚本
- 连接远程主机
- 使用 `rsync` / `ssh` 配合远程 LSP

## 打开终端

终端通常会以独立页面打开，并尽量带上当前项目路径。

常见用法：

- 从项目或编辑工作流进入终端
- 在现有终端页面里新建会话
- 针对当前项目打开对应工作目录的终端

## 多会话与标签

当前终端支持：

- 多个终端会话并存
- 会话切换
- 关闭会话
- 重新启动失败的会话
- 为不同项目恢复各自的终端状态

如果你经常在多个项目间切换，这一点会很有用。

## 终端后端

新建终端时，当前实现支持选择不同后端：

- **HOST** - 更接近宿主环境
- **PROOT** - 适合完整 Linux / 工具链场景

如果你要使用更完整的命令行环境、包管理和调试链路，通常优先考虑 **PROOT**。

## Shell 支持

当前设置中可配置 Shell 类型，常见场景为：

- **Bash** - 默认、最稳妥
- **Zsh** - 可选，适合已有 Zsh 使用习惯的用户

如果切换 Shell 后无法正常启动，先回到 Bash 验证基础环境是否正常。

## 输入方式

### 扩展按键栏

终端页面提供额外按键区，方便输入：

- `Ctrl`
- `Alt`
- `Esc`
- 其他常用控制键

### 硬件键盘

如果你连接了硬件键盘，常见组合键依旧有价值，例如：

- `Ctrl + C` - 中断当前命令
- `Ctrl + L` - 清屏
- `Tab` - 自动补全

## 项目内常见工作流

### CMake / Make

```bash
cmake -S . -B build
cmake --build build
```

或：

```bash
make
```

### Git

```bash
git status
git add .
git commit -m "update"
```

### 远程协作

```bash
ssh user@host
rsync -av ./ user@host:/path/
```

## 会话恢复

当前实现支持按项目路径保存和恢复终端状态，因此：

- 重进同一个项目时，终端状态通常可以延续
- 多个项目的终端会话彼此相对独立

## 常见问题

### 终端启动失败

优先检查：

- 开发环境是否部署完整
- PRoot 相关资源是否可用
- 是否需要在设置中重新部署开发环境

### 中文输入不顺手

部分输入法在终端中的兼容性仍可能一般。更稳定的方式通常是：

- 使用更兼容的输入法
- 连接硬件键盘
- 先在编辑器中输入，再复制到终端

### 命令能在一个项目里跑，在另一个不行

通常是因为：

- 当前终端工作目录不同
- 该项目的构建目录 / 环境变量还没准备好
- 启动时使用了不同后端

## 建议用法

如果你主要做图形化编辑，终端最好承担这类任务：

- 构建脚本
- Git 高级操作
- 远程连接
- 辅助调试

这样能让编辑器和命令行各自发挥长处。

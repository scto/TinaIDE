# 编辑器基础

本文基于当前工作台实现整理，描述的是 MobileIDE 现在的编辑体验，而不是传统桌面 IDE 的菜单式布局。

## 工作台布局

典型的编辑工作台可理解为：

`	ext
┌──────────────────────────────────────┐
│ 顶部操作：保存 / 运行 / 构建 / 调试  │
├──────────────┬───────────────────────┤
│ 文件树抽屉   │ 标签栏                 │
│ 项目目录     ├───────────────────────┤
│              │ 编辑区域               │
│              │ 多标签切换             │
├──────────────┴───────────────────────┤
│ 底部面板：构建日志 / 诊断 / 大纲 /    │
│          符号 / 书签 / Git           │
└──────────────────────────────────────┘
`

终端不是常驻在底部面板里的主要标签，而是作为 **独立终端页面** 使用。

## 打开文件与切换标签

当前最常用的文件操作方式有：

- 从 **文件树** 打开文件
- 在 **标签栏** 间快速切换多个已打开文件
- 关闭当前标签或关闭全部标签

如果你使用硬件键盘，还可以在 **设置 → 快捷键** 中自定义标签切换与关闭行为。

## 常用编辑动作

当前工作台最稳定、最高频的编辑能力包括：

- **保存当前文件 / 保存全部**
- **撤销 / 重做**
- **项目内搜索与替换**
- **书签切换与跳转**
- **格式化文档**
- **多标签编辑**

## 代码智能能力

在语言服务工作正常时，编辑器可以提供这些能力：

- 代码补全
- 语义诊断
- 跳转到定义
- 查找引用
- 符号与大纲视图
- 部分场景下的快速修复 / 重命名

如果这些能力没有生效，优先检查：

- 项目是否已经成功构建过
- 是否存在可用的 compile_commands.json
- clangd / 远程 LSP 配置是否正常

## 底部面板怎么看

### 构建日志

用于查看：

- 编译输出
- 链接错误
- 运行前后的构建细节

### 诊断

用于查看：

- clangd 上报的错误与警告
- 点击后可快速定位到代码位置

### 大纲 / 符号

适合在单个文件里快速定位：

- 函数
- 类
- 结构体
- 重要符号节点

### 书签

适合在长文件或来回修改多个位置时快速跳转。

### Git

可查看工作区变更、暂存状态等内容，是编辑工作流中很常用的辅助面板。

## 文件查看器

当前工作台已经明确接入的查看器类型包括：

- **JSON**
- **大文件文本**
- **图片**
- **Hex**

这类文件会按更合适的查看方式打开，而不是强行走普通代码编辑模式。

## 与终端的配合方式

如果你要执行以下操作，建议切换到独立终端：

- 手写 cmake / make 命令
- 运行 Git 命令
- 执行脚本
- 进行 SSH / rsync / 远程开发相关操作

## 建议的上手顺序

1. 学会用文件树打开文件
2. 熟悉标签切换
3. 观察构建日志与诊断面板
4. 熟悉书签与符号跳转
5. 再进一步学习终端、调试与 Git

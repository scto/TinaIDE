# 插件设置说明

本文基于当前插件设置页和相关插件页面源码整理，重点说明已安装插件管理、从文件安装、插件日志，以及主题 / LSP 插件相关能力。

## 先理解当前插件入口

当前 **设置 → 插件** 更像是 **已安装插件管理页**，不是浏览全量插件目录的总入口。

现在直接提供的能力主要包括：

- 查看已安装插件列表
- 进入插件详情
- 管理模式批量选择 / 卸载
- 从文件安装 `.mobileplug`
- 查看插件日志

如果你是想装一个本地插件包，这里就够用了；如果你是想浏览更多新插件，通常要走首页 **市场** 或其他独立入口，而不是依赖当前设置页顶部菜单。

## 列表页能看到什么

当前列表会展示这些关键信息：

- 插件名称和版本
- 已禁用标记
- 内置插件标记
- 主题插件标记
- LSP 插件就绪 / 未就绪状态

如果当前没有已安装插件，页面会直接显示空状态提示。

## 顶部更多菜单

当前插件列表右上角更多菜单里直接有三项：

- **管理插件**
- **从文件安装插件**
- **插件日志**

注意一点：

- 当前设置页顶部菜单里**没有直接“插件市场”按钮**

所以旧文档如果把“插件市场”写成当前设置页的直接入口，那已经不符合现在的实现。

## 插件详情页能做什么

进入单个插件后，当前详情页主要提供：

- 启用 / 禁用开关
- 描述、插件 ID、版本、类型
- 最低应用版本、作者、仓库、许可证、安装目录
- 贡献摘要
  - 主题数量
  - 文件树菜单扩展数量
  - 编辑器右键菜单扩展数量

## 主题型插件

如果插件带有编辑器主题，详情页会出现 **插件主题** 入口。

选择后会直接把当前编辑器主题切到该插件主题。

这也是为什么有些主题并不只出现在内置主题列表里，而是通过插件提供。

## LSP 插件

如果插件同时声明了 LSP 工具链依赖，详情页会出现依赖安装入口。

当前界面会区分：

- **已就绪**
- **未就绪**

如果语法高亮、补全或语言扩展已经装上了却没有真正工作，先检查是不是还没把对应 LSP 依赖装完整。

## 从文件安装怎么工作

当前设置页会调用系统文件选择器导入插件文件。

### 支持的包形式

字符串说明里明确写的是：

- `.mobileplug`（zip）文件

### 脚本 / 混合插件的权限确认

如果插件属于 `script` 或 `hybrid` 类型，并且声明了需要用户确认的权限，安装前会先弹权限确认对话框。

也就是说，当前实现不会把这类插件静默装进去。

如果你点了拒绝，临时安装文件也会被清理。

## 管理模式和卸载

进入 **管理插件** 后，当前页面支持：

- 多选
- 全选 / 取消全选
- 批量卸载

### 内置插件的限制

源码里明确限制：

- **内置插件不能卸载**

如果你只选中了内置插件，当前会给出提示，而不是执行卸载。

### 单个插件卸载

在插件详情页里也可以直接卸载单个插件。

但同样，内置插件不会被真的卸掉。

## 插件日志能做什么

插件日志页当前支持：

- 按日志级别筛选
- 按插件筛选
- 关键词搜索
- 自动滚动开关
- 查看单条日志详情
- 复制日志内容
- 导出日志
- 清空全部日志或指定插件日志

如果你怀疑某个插件启用后行为异常，这里比盲猜更有用。

## 什么时候优先看插件设置

适合场景：

- 本地安装 `.mobileplug` 插件包
- 想确认插件到底有没有启用
- 想切换插件提供的主题
- LSP 插件装了但还没准备好
- 想排查某个插件报错或权限问题

## 常见问题

### 为什么插件装上了却没看到效果

先检查：

1. 插件是否真的处于启用状态
2. 如果是主题插件，是否已经在插件详情或编辑器设置里切过去
3. 如果是 LSP 插件，是否还缺依赖工具链
4. 插件日志里有没有报错

### 为什么某些插件删不掉

因为当前实现禁止卸载内置插件。

### 为什么从文件安装后还会弹权限确认

因为 `script` / `hybrid` 插件会在安装前先做权限确认，这属于当前实现的正常流程。

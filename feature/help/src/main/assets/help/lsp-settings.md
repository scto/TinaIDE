# LSP 设置说明

本文基于当前 **设置 → 语言服务器** 页面源码整理，重点说明 clangd、本地 LSP 行为，以及远程 LSP 配置里真正会出现的选项。

## 当前 LSP 设置页分成哪几块

现在的页面主要分成三组：

- **Clangd 配置**
- **LSP 行为设置**
- **远程 LSP 服务器**

如果你只是觉得“补全不准”或“远程连接不稳定”，先把问题归到这三组里的某一组，再去排查会更快。

## Clangd 配置

这一组主要控制本地 clangd 的工作方式。

当前会出现这些项目：

- 运行模式
- 后台索引
- Clang-Tidy
- 头文件自动插入
- 补全信息详细程度
- 函数参数占位符

### 运行模式

当前实现里：

- **native**
- **proot**

其中 `proot` 只有在 Linux 开发环境已启用时才会出现在设置页。

怎么理解：

- **native**：性能更好
- **proot**：兼容性更好

如果你在本地 clangd 上遇到奇怪的环境差异，`proot` 值得试；如果你更在意设备性能，通常先留在 `native`。

### 后台索引

这个开关决定 clangd 是否做跨文件分析用的后台索引。

一般效果是：

- **开**：跨文件补全和跳转更准，但更吃资源
- **关**：更省资源，但大型项目体验会差一些

### Clang-Tidy

开启后会让 clangd 提供额外的静态分析警告和改进建议。

如果你只是想先把补全跑顺，不一定要第一时间开它。

### 头文件自动插入

当前只有两个值：

- **智能插入 (IWYU)**
- **关闭**

如果你不喜欢补全时顺手带出 `#include` 建议，可以直接关掉。

### 补全信息详细程度

当前只有两个值：

- **详细**
- **简洁**

如果你希望候选里看到更多上下文，选“详细”；如果你只想让列表干净一点，选“简洁”。

### 函数参数占位符

这个开关控制函数补全后是否顺手插入参数占位符。

如果你喜欢一路 Tab 填参数，开着更方便；如果你觉得编辑区里总是多出一堆占位文本，可以关掉。

### 修改后什么时候生效

当前页面本身就给了提示：

- 修改后通常需要 **重新打开文件** 或 **重启 LSP**

所以改完后如果你立刻没看到变化，不一定是设置没保存。

## LSP 行为设置

这一组主要控制编辑器怎么展示和使用 LSP 返回的数据。

当前会出现这些项目：

- 补全候选数量
- 大小写敏感
- 参数提示窗口
- 行内参数提示
- 语义高亮
- 折叠范围

### 补全候选数量

当前是滑块，不是输入框。

有效范围是：

- `10`
- 到
- `200`

值越大，clangd 一次返回的候选越多，也越容易消耗资源。

如果你在大项目里觉得补全弹出很慢，先尝试把这个值调低，而不是一上来怀疑全部 LSP 都坏了。

### 大小写敏感

这个开关比较直接：

- **开**：更严格
- **关**：更宽松

如果你经常记不清符号大小写，关闭会更友好。

### 参数提示窗口

当前会在输入 `(` 或 `,` 这类场景下显示函数签名和当前参数位置。

如果你觉得输入时弹窗太频繁，可以关掉。

### 行内参数提示

这就是 Inlay Hints。

特点是：

- 只显示在行内
- 不会真正写进文件文本

如果你想看得更清楚但又不想污染源代码，这项很有用。

### 语义高亮和折叠范围

这两项在当前字符串里都标成了：

- **实验**

也就是说，它们更适合你愿意尝试新效果时打开；如果你追求稳定，先保持默认更稳。

## 远程 LSP 服务器

这一组用于把语言分析搬到远端设备。

当前会出现这些项目：

- 启用开关
- 主机
- 端口
- 远程工作区根路径
- 同步模式
- 同步方法
- rsync 模块
- rsync 端口
- 连接状态
- 测试连接

### 启用开关

只有启用远程 LSP 后，测试按钮才会真正可用。

如果你没开这个开关，就算主机和端口都填了，也不算处于远程 LSP 工作状态。

### 主机

当前主机输入有几个限制：

- 不能带空格
- 不要填 `ws://`
- 不要填 `wss://`
- 不要填 `http://` 或 `https://`

也就是说，这里应该填：

- `192.168.1.10`
- `devbox.local`
- `localhost`

而不是完整 URL。

当前实现里如果你填的是 `localhost` 或 `127.0.0.1`，还会给出额外提示，因为这类地址在移动端远程联机场景里很容易填错目标。

### 端口

当前默认值是：

- `6789`

有效范围是：

- `1..65535`

### 远程工作区根路径

这个值可以留空。

如果你要填，当前校验接受的形式主要是：

- `file://...`
- Windows 绝对路径
- 以 `/` 开头的绝对路径

它更适合：

- **MANUAL**
- **RSYNC**

这类需要你明确远端项目目录的场景。

### 同步模式

当前有三个值：

- **AUTO**
- **LIGHTWEIGHT**
- **PROJECT**

怎么理解：

- **AUTO**：先让应用自动判断，当前也是最推荐的起点
- **LIGHTWEIGHT**：偏向只传当前打开文件，适合更轻量的远程协作
- **PROJECT**：偏向完整项目上下文

如果当前是 `AUTO`，而应用已经检测出具体模式，页面还会额外显示：

- 检测到的模式
- 检测原因

### 同步方法什么时候会出现

这一点很容易忽略：

- **同步方法** 不是总会显示

当前只有在下面两种情况之一时才显示：

1. 你手动选了 `PROJECT`
2. 你选的是 `AUTO`，但自动检测结果落到了 `PROJECT`

所以如果你发现页面里看不到“同步方法”，不一定是 bug，通常只是你当前并不在项目模式。

### 同步方法

当前有三个值：

- **BUILTIN**
- **RSYNC**
- **MANUAL**

一般可以这样理解：

- **BUILTIN**：零配置起步，适合小中型项目
- **RSYNC**：增量同步，适合大项目
- **MANUAL**：同步动作交给你自己控制

### rsync 配置

只有在同步方法选成 `RSYNC` 时，页面才会继续显示：

- rsync 模块
- rsync 端口

当前默认值分别是：

- 模块：`mobile-workspace`
- 端口：`873`

模块名当前还有限制：

- 不能为空
- 不能带空格
- 不能带 `/`

### 连接状态和测试连接

页面会显示这些状态之一：

- `DISCONNECTED`
- `CONNECTING`
- `CONNECTED`
- `ERROR`

点 **测试连接** 时，当前会去尝试建立 WebSocket 连接。

如果测试成功：

- 页面会把连接状态更新到 `CONNECTED`

如果主机没填，当前会直接提示配置无效，而不是开始测试。

## 什么时候该怎么配

### 本地小项目

一般先这样就够：

- clangd 运行模式：`native`
- 后台索引：按资源情况决定
- 远程 LSP：关闭

### 本地大项目

优先尝试：

- 保证 `compile_commands.json` 正常
- 视情况打开后台索引
- 适当降低补全候选数量

### 远程大项目

建议从这个顺序开始：

1. 开启远程 LSP
2. 填主机和端口
3. 同步模式先用 `AUTO`
4. 测试连接
5. 如果工程很大，再考虑 `PROJECT + RSYNC`

## 常见问题

### 为什么“运行模式”里没有 PRoot

因为当前只有在 Linux 开发环境启用时，设置页才会把 `proot` 作为 clangd 运行模式选项显示出来。

### 为什么“同步方法”不见了

因为当前只有项目模式下才显示同步方法。

### 为什么测试连接老是失败

优先检查：

1. 远程 LSP 开关是否已启用
2. 主机是不是写成了完整 URL
3. 端口是不是和远端服务一致
4. 是否错误地把电脑地址写成了移动端自己的 `localhost`

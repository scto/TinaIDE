## MobileIDE 国际化（i18n）统一规范

目标：
- 字符串统一通过资源 id 获取，避免到处 `import R`、避免到处传 `Context`
- 仍使用 `@StringRes Int` + `getString(id)`，不使用 `getIdentifier()` 等动态查找（性能/安全更好）

### 1) 初始化（必须）

在 `Application.onCreate()` 初始化一次全局上下文：
- `app/.../MobileApplication.kt` 调用 `AppStrings.initialize(this)`
- 实现位于 `core/i18n/.../core/i18n/AppStrings.kt`

### 2) 字符串资源的统一入口

资源别名（避免 `R.string.xxx`）：
- `core/i18n/.../core/i18n/TextResourceAliases.kt`
  - `typealias Strings = R.string`
  - `typealias Arrays = R.array`
- `app/.../core/i18n/AppResourceAliases.kt`
  - `typealias Drawables = R.drawable`

字符串扩展：
- `core/i18n/.../core/i18n/ResExt.kt`
  - `Strings.xxx.str(...)`：使用全局 `AppStrings`（不需要传 `Context`）
  - `Strings.xxx.strOr(context, ...)`：优先使用传入 `Context`（如需要 Activity 的 locale wrapper）
  - `Arrays.xxx.stringArray()` / `Arrays.xxx.stringArrayOr(context)`：读取 `string-array`

注意：
- `str/strOr` 支持可空参数（会将 `null` 安全替换为空字符串），实现位于 `AppStrings.get/getOr`
- 各 feature 模块可定义自己的资源别名文件（如 `feature/settings/.../core/i18n/SettingsResAliases.kt`）

### 3) Compose 用法（推荐）

- 直接用 `stringResource(Strings.xxx, ...)`
- 资源 id 使用所在模块实际提供的别名，例如 app 层 `Drawables.xxx`、settings 模块 `Raw.xxx`

### 4) 非 Compose 用法（推荐）

- 有上下文：`Strings.xxx.strOr(context, ...)`
- 无上下文：`Strings.xxx.str(...)`（依赖 `AppStrings.initialize`）

### 5) 大段文本（协议/隐私政策等）

不要在 Kotlin 里写多行中文常量；放到所属模块的 `res/raw`、`assets` 或后端可维护内容源，并按需要提供语言变体：
- 资源型大文本优先放在对应 feature/core 模块的 `res/raw`
- 帮助中心内容当前由 `feature/help/src/main/assets/help/*.md` 维护

### 6) 迁移检查清单

- UI：`stringResource(Strings.xxx)`，不要 `stringResource(R.string.xxx)`
- 业务/核心：不要 `context.getString(R.string.xxx)`，用 `Strings.xxx.strOr(context)` 或 `Strings.xxx.str()`
- XML（menu/layout/preferences）：不要硬编码 `android:title="中文"` / `android:text="中文"`，用 `@string/xxx`
- 字符串不要硬编码中文（尤其是 UI/Toast/错误提示）；优先进入 `strings.xml` 或 `res/raw-*`
- 新增 key 时同时补 `values/strings.xml` 与 `values-en/strings.xml`（至少给出英文/占位）

### 7) 自动化脚本（推荐）

在 `tools/i18n/` 下提供了维护脚本：
- `python tools/i18n/check_all.py`：一键验收（values-en 同步 + Kotlin 字符串字面量扫描 + res XML 属性硬编码扫描）
- `python tools/i18n/check_all.py --include-logs`：更严格（包含日志行）
- `python tools/i18n/sync_values_en.py`：将缺失 key 追加到 `values-en/strings.xml`（默认复制中文作为占位，后续可逐步翻译）
- `python tools/i18n/sync_values_en.py --check`：仅检查缺失 key（用于 CI/本地验收）
- `python tools/i18n/fix_mojibake.py --apply`：尝试修复常见乱码（mojibake）
- `python tools/i18n/check_hardcoded_cjk.py`：扫描 Kotlin 字符串字面量中的硬编码中文

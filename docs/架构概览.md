# MobileIDE 架构概览

> 更新日期：2026-04-22

MobileIDE 当前采用 `app + core/* + feature/* + tools/*` 的模块化结构，依赖注入基于 Koin。
和旧文档最大的区别仍然是：**默认编译 / LSP 链路已经切到 native toolchain + Android sysroot，PRoot 退回为可选 Linux 环境。**

## 分层总览

```text
┌─────────────────────────────────────────────────────────────┐
│ UI Layer                                                   │
│ MainPortalActivity / MainActivity / Compose screens        │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│ App & Feature Layer                                         │
│ app + feature:editor / ai / help / terminal / settings ...  │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│ Core Layer                                                  │
│ compile / ndk / lsp / editor-lsp / plugin / project ...     │
└──────────────────────┬────────────────┬─────────────────────┘
                       │                │
         ┌─────────────▼───────┐   ┌────▼─────────────────────┐
         │ Native Runtime       │   │ Optional Linux Runtime   │
         │ mobile-toolchain       │   │ PRoot + linux-distro    │
         │ Android sysroot      │   │ guest toolchain          │
         │ native clangd / run  │   │ terminal / plugin tools  │
         └──────────────────────┘   └──────────────────────────┘
```

## 关键边界

### 入口结构

当前 App 入口已经分成两层：

- `MainPortalActivity`：主页 / 项目入口 / 设置跳转
- `MainActivity`：打开项目后的主编辑器工作区

不要再把“首页”和“项目编辑器”理解成同一个 Activity。

### 默认编译链路

基础编译、运行、clangd/LSP 依赖的是：

- `AndroidNativeToolchainManager`
- `AndroidSysrootManager`
- `CompileProjectUseCase`
- `NativeClangdConnectionProvider`

也就是说，默认开发链路不要求 PRoot 先安装完成。

### 编辑器语言服务分流

`LspEditorManager` 当前会按文件类型和配置做四类分流：

1. C / C++：走 `LspClientSession`，后端可选 native clangd、PRoot clangd、remote LSP
2. CMake：走 `CMakeLanguageServiceSession`
3. Make / `.mk`：走 `MakeLanguageServiceSession`
4. 其他语言：若存在匹配插件，则走 `LspPluginManager + PluginLspConnectionProvider`

这意味着“编辑器语言服务”已经不是 `clangd` 单一路径。

### AI 工具体系

`feature:ai` 当前通过 `ToolInitializer` 向 `ToolRegistry` 注册内置工具，按类别组织：

- 编辑器
- 文件系统
- 代码分析
- 诊断
- 构建
- 执行
- 重构
- Git
- 终端
- Web
- Custom

原先零散写成“项目管理 / Web / GitHub 搜索”的能力，当前分别落在
`FILE_SYSTEM`、`CODE_ANALYSIS`、`GIT`、`TERMINAL`、`WEB` 等分类中。

### 可选 Linux 环境

`core:proot` 负责的是可选 Linux 环境：

- 自研 Linux 发行版安装
- PRoot 生命周期
- Linux 终端与 guest toolchain

它主要服务：

- Linux shell / 命令行场景
- 某些依赖 Linux 侧工具的插件 / 扩展能力
- 部分调试链路

### 启动安装流程

首次启动时，`StartupFlowManager` 默认会拉起依赖安装页，并且：

- 默认只安装 `Android sysroot + native toolchain`
- 不自动安装 Linux 发行版 / PRoot

只有显式进入 Linux 环境入口时，`DependencyInstallViewModel` 才会追加：

- Linux distro runtime metadata
- `Linux rootfs`
- `PRoot guest toolchain`

## 核心流程

### 1. 首次启动

1. `MainPortalActivity` 启动前调用 `StartupFlowManager.checkStartupFlow()`
2. 检查 `WorkspaceSetupCompleted`
3. 检查 `Android sysroot` 与 `native toolchain` 是否就绪
4. 未就绪时进入 `DependencyInstallActivity`
5. `DependencyInstallViewModel` 安装并校验运行资产

### 2. 打开项目

1. 用户在 `MainPortalActivity` 选择项目
2. `IProjectSession` 打开项目并注册路径
3. 跳转到 `MainActivity`
4. 编辑器、文件树、终端、底部面板开始装配

### 3. 编译 / 运行

1. UI 触发编译动作
2. `CompileProjectUseCase` 识别项目类型与运行配置
3. 对应 `BuildStrategy` 调用 native toolchain 或 PRoot 侧工具
4. `ProgramRunner` 负责运行产物或准备终端命令

### 4. 编辑器语言服务

1. 打开文件后，`LspEditorManager` 判断文件类型与当前配置
2. C/C++ 走 `LspClientSession`，按设置选择 native / PRoot / remote
3. CMake / Make 走内建语言服务会话
4. 插件语言走 `LspPluginManager`
5. 诊断、补全、跳转、语义 token 同步到底部面板和编辑器状态

### 5. 可选 Linux 环境安装

1. 用户在设置页或工作区入口显式发起安装
2. PRootBootstrap 调用 SelfHostedLinuxDistroRuntime 安装发行版 rootfs
3. `DependencyInstallViewModel` 继续安装 guest toolchain、sysroot、native toolchain
4. Linux 终端与相关功能可用

## 当前最容易误解的点

- PRoot 不是默认编译器宿主环境。
- “首次启动安装 rootfs” 已经不是默认事实；默认安装的是 sysroot 与 native toolchain。
- 编辑器语言服务不是只有 clangd；CMake / Make 已有内建语言服务，其他语言还可以走插件 LSP。
- Linux rootfs 来源由 `tools/linux-distro` manifest 描述，不再通过旧静态 Docker rootfs 资产包注入。

# TinaIDE 快速开始

> 更新日期：2026-04-06

这份指南面向当前仓库版本，重点说明 TinaIDE 的真实启动链路和构建入口。

## 先理解运行时结构

TinaIDE 当前有两套相关但独立的运行资产：

- 默认必需：`Android sysroot` + `native tina-toolchain`
  用于编译、运行、C/C++ clangd，也是首次启动默认安装的内容。
- 可选扩展：`Linux distro rootfs` + `PRoot guest toolchain`
  用于 Linux 终端环境、插件 LSP 所需的 Linux 命令、Linux 侧工具链路，不是基础编译的前置条件。

如果你只是想把 App 跑起来并编译 C/C++ 项目，先关注第一套即可。

## 环境准备

### 必需工具

| 工具 | 版本建议 | 说明 |
|------|----------|------|
| Android Studio | 最新稳定版 | 打开工程、调试、安装 APK |
| JDK | 17+ | Gradle / Android 构建 |
| Git | 任意稳定版 | 拉取仓库与子模块 |

### 可选工具

| 工具 | 何时需要 |
|------|----------|
| PowerShell 7+ | 使用仓库自带脚本时体验更好 |
| Docker Desktop | 只有维护者重建 toolchain / sysroot / PRoot 运行资产时需要 |

## 克隆项目

```bash
git clone --recursive https://github.com/wuxianggujun/TinaIDE.git
cd TinaIDE
```

如果已经克隆过主仓库但没带子模块：

```bash
git submodule update --init --recursive
```

## 构建 App

### 方式一：使用仓库脚本（推荐）

```powershell
# 默认构建 arm64 Debug，并安装到设备
pwsh ./tools/build-apk.ps1 -Install

# 构建 x86_64 Debug，用于模拟器
pwsh ./tools/build-apk.ps1 -Abi x86 -Install

# 构建 arm64 Release
pwsh ./tools/build-apk.ps1 -Variant release

# 构建所有 ABI 的 Release
pwsh ./tools/build-apk.ps1 -Variant release -AllAbi
```

说明：

- `-Abi arm64` 对应 `arm64-v8a`
- `-Abi x86` 对应 `x86_64`
- `-AllAbi` 会调用 `assembleDebugAllAbi` / `assembleReleaseAllAbi`
- Release 构建前需要准备本地 `keystore.properties` 和签名文件
- 可先复制 `keystore.properties.example` 为 `keystore.properties`

### 方式二：直接用 Gradle（高级用法）

```bash
# 默认本地开发只启用一个 ABI（默认 arm64）
./gradlew :app:assembleArm64Debug
./gradlew :app:installArm64Debug

# 切到 x86_64 本地开发 ABI
./gradlew "-Ptina.devAbi=x86_64" :app:assembleX86_64Debug
./gradlew "-Ptina.devAbi=x86_64" :app:installX86_64Debug

# 构建所有 ABI 的 Debug / Release
./gradlew :app:assembleDebugAllAbi
./gradlew :app:assembleReleaseAllAbi
```

当前工程启用了按 ABI 的 product flavor，并默认使用“单 ABI 本地开发模式”：

- 默认 `arm64`
- 通过 `"-Ptina.devAbi=x86_64"` 切换到模拟器 ABI
- 通过 `-Ptina.allAbi=true` 或 `assemble*AllAbi` 恢复双 ABI 构建

在 PowerShell 下，`-P` 参数建议始终加引号，避免被错误解析。

## 首次启动会发生什么

首次启动或运行资产损坏时，`StartupFlowManager` 会跳到依赖安装页。默认流程会安装：

1. `Android sysroot`
2. `native tina-toolchain`

默认不会自动安装 Linux rootfs。

安装完成后，你就可以：

1. 在 `MainPortalActivity` 里创建或打开项目
2. 进入 `MainActivity` 编辑器
3. 对单文件 / CMake / Make 项目进行构建与运行
4. 在 C/C++ 文件里使用 clangd 获取补全与诊断

## 什么时候需要安装 Linux distro / PRoot

只有你确实需要 Linux 终端环境、Linux 侧工具或插件 LSP 依赖的 Linux 命令时，才需要继续安装自研 Linux distro / PRoot 环境。

常见入口：

- 设置页的 Linux / 存储相关入口
- 工作区依赖安装入口

此时安装流程会额外包含：

1. `Linux distro rootfs`
2. `Linux distro runtime metadata`
3. `PRoot guest toolchain`
4. 然后再校验 / 安装 `Android sysroot`
5. 然后再校验 / 安装 `native tina-toolchain`

## 创建并运行第一个项目

1. 启动 TinaIDE
2. 在首页选择或创建项目
3. 选择单文件、CMake 或 Make 模板
4. 打开 `main.cpp`、`CMakeLists.txt` 或 `Makefile`
5. 点击运行 / 构建按钮

当前语言服务大致分为三类：

- C/C++：clangd
- CMake / Make：内建语言服务
- 其他语言：如果安装了匹配插件，可走插件 LSP

## 可选：维护者重建运行资产

大多数开发者不需要执行这一节。

### 重建 PRoot 运行时 / sysroot

```powershell
pwsh ./docker/proot-build/build-proot.ps1 -CopyToJniLibs -CopyToAssets
pwsh ./docker/proot-build/build-proot.ps1 -BuildSysroot
```

默认构建会从 `external/termux-proot` 源码直接编译 PRoot：

```powershell
.\gradlew.bat :app:assembleArm64Debug
```

如需临时使用 Docker 产出的预编译 PRoot，可关闭源码直编：

```powershell
.\gradlew.bat :app:assembleArm64Debug '-Ptina.buildProotFromSource=false'
```

预编译产物目录为 `app/src/prebuiltProot/jniLibs`。

### 同步 tina-toolchain / sysroot 资产

当前 native 工具链资产采用 `current.properties + .tar.xz` 的规格化布局。
维护者在更新工具链 release 产物后，应同步执行：

```powershell
pwsh ./tools/sync-tina-toolchain-assets.ps1 -Abi arm64 -Clean
pwsh ./tools/sync-tina-toolchain-assets.ps1 -Abi x86_64 -Clean
```

相关事实源：

- `app/src/<abi>/assets/tina-toolchain/current.properties`
- `tools/sync-tina-toolchain-assets.ps1`
- `app/build.gradle.kts` 中的 `verifyTinaToolchainAssets`

### 刷新自研 Linux 发行版 manifest

```powershell
pwsh ./tools/linux-distro/generate-linux-distro-manifest.ps1
```

注意：

- TinaIDE 不再通过静态 rootfs 资产包注入默认 Linux 环境。
- `tools/linux-distro` 维护发行版 manifest 与锁定元数据。
- App 代码中的 Linux 环境安装器走 `core:linux-distro` + `SelfHostedLinuxDistroRuntime`。

## 常见问题

### Q: 首次启动一直停在依赖安装页

先区分是哪一类依赖：

- 默认启动流程：安装的是 `android-sysroot` 与 `native toolchain`
- Linux 环境流程：会额外安装 `Linux distro rootfs`、`Linux distro runtime metadata` 与 `PRoot guest toolchain`

建议检查：

```bash
adb logcat -s StartupFlowManager NativeToolchainManager AndroidSysrootManager PRootBootstrap
```

### Q: 想在模拟器上跑，应该构建哪个 ABI

```powershell
pwsh ./tools/build-apk.ps1 -Abi x86 -Install
```

或：

```bash
./gradlew "-Ptina.devAbi=x86_64" :app:assembleX86_64Debug
```

### Q: Release 构建失败

通常是签名配置缺失。请确认：

- 已从 `keystore.properties.example` 复制出本地 `keystore.properties`
- `storeFile` 指向的本地 keystore 文件存在
- `storeFile`、`storePassword`、`keyAlias`、`keyPassword` 配置正确

另外，Release 任务会有额外副作用：

- 自动递增 `version.properties`
- 备份 mapping 文件

mapping 文件仅由公开构建逻辑做本地归档。若维护者需要外部 symbol 存储集成，
应放在公开仓库之外维护。

### Q: clangd 没有补全

优先检查：

1. 工具链是否安装完成
2. `compile_commands.json` 是否存在
3. 当前文件是不是 C/C++，而不是 CMake / Make
4. 是否切到了 remote / PRoot / 插件 LSP 路径

调试日志：

```bash
adb logcat -s LspEditorManager NativeClangd PRootClangd RemoteLsp PluginLspConnection LspPluginManager
```

### Q: 我需要 Linux 终端，但当前只有 native toolchain

进入设置页或工作区依赖安装页，显式安装 Linux distro / PRoot 环境即可。它和基础编译链路是拆开的。

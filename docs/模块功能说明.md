# MobileIDE 模块功能说明

> 更新日期：2026-04-29
>
> 本文基于 `settings.gradle.kts`、各模块 `build.gradle.kts` 与
> `src/main` 源码目录梳理，覆盖主工程 `include(...)` 模块、本地外部模块、
> `tools/*` 模板模块，并补充 `includeBuild(...)` 复合构建。

## 总体分层

MobileIDE 当前是典型的 `app + feature/* + core/* + external/* + tools/*`
多模块结构，运行期装配主要由 `:app` 负责，业务页面放在 `feature/*`，
可复用能力与运行时能力沉淀到 `core/*`。

```text
:app
├── feature/*    用户可见功能页与 ViewModel
├── core/*       基础能力、领域模型、编译/LSP/插件/运行时能力
├── external/*   本地化维护的第三方或 JNI 运行时依赖
└── tools/*      新建工程模板中的可复用 Android 模块
```

依赖方向上，`feature/*` 只消费 `core/*` 与必要外部 UI 库；`core/*` 尽量沉淀
无界面的通用能力；`external/*` 保持第三方源码边界；`tools/*` 面向被生成或
被打包的模板工程，不承载主 App 业务。

## App 壳模块

- `:app`（`app`）
  - 功能：应用启动、Koin 装配、主页入口、项目工作区入口、全局导航、编译运行 UI 桥接、终端/搜索/Git/调试/插件等跨模块协调。
  - 主要源码：`MobileApplication.kt`、`MainPortalActivity.kt`、`MainActivity.kt`、`startup/*`、`ui/*`、`di/*`。
  - 说明：它是集成层，不应把具体领域逻辑继续堆入 `:app`；新增独立业务优先下沉到 `feature/*` 或 `core/*`。

## Core 基础模块

- `:core:cmake`（`core/cmake`）：CMake 脚本模型、命令/变量/目标解析、生成器表达式与语义诊断，支撑 CMake 项目分析、补全和构建策略。

- `:core:common`（`core/common`）：跨模块公共接口与工具集合，包含导航抽象、项目/文件操作接口、差异模型、终端抽象、通用扩展和服务生命周期。

- `:core:config`（`core/config`）：应用偏好、主题、快捷键、LSP/clangd、AI 渠道、许可证和服务端配置等全局配置入口。

- `:core:database`（`core/database`）：Room 用户内容数据库，管理收藏、下载历史、用户内容本地仓储，以及收藏同步 Worker。

- `:core:designsystem`（`core/designsystem`）：Compose 设计系统，提供主题颜色、字体、间距、形状、动画、语义色与通用 UI 状态组件。

- `:core:i18n`（`core/i18n`）：国际化字符串访问封装，提供 `AppStrings`、资源别名和字符串扩展，统一 UI 文案读取方式。

- `:core:logging`（`core/logging`）：Timber 日志树、文件日志、手势追踪、日志导出清单与隐私分级策略。

- `:core:model`（`core/model`）：跨模块领域模型，包含项目类型、文件节点、C/C++ 标准、Android API、项目元数据和反馈模型。

- `:core:network`（`core/network`）：网络基础设施，包含 OkHttp 客户端、智能 DNS、DoH、API 结果封装、错误分类和用户内容 API 客户端。

- `:core:security`（`core/security`）：安全校验能力，包含路径验证与服务端配置 HMAC 验证，防止越权路径和伪造配置进入核心流程。

- `:core:storage`（`core/storage`）：项目存储与外部文件能力，包含项目路径、导入导出、清理、权限请求、项目位置数据库和外部文件 Intent。

- `:core:text-engine`（`core/text-engine`）：编辑器文本底座，包含 Rope 文本缓冲、行索引、编辑历史、文本扫描 native bridge 和位置模型。

- `:core:tree-sitter`（`core/tree-sitter`）：Tree-sitter 高亮适配层，提供增量语法树、高亮模型、折叠区域、查询加载/编译和语言注册表。

- `:core:editor-view`（`core/editor-view`）：Compose 编辑器渲染与交互视图，覆盖光标、选择手柄、补全弹窗、诊断波浪线、括号引导线、折叠与可视区域渲染。

- `:core:editor-lsp`（`core/editor-lsp`）：编辑器与 LSP 间的轻量契约，包含补全、诊断、导航、符号、语义 token 和 hover 等 UI 可消费接口。

## Core 中间能力模块

- `:core:apk-builder`（`core/apk-builder`）：APK 打包、模板补丁、图标栅格化、manifest 权限覆写、zipalign、签名和调试 keystore 管理。

- `:core:compile`（`core/compile`）：编译与运行主流程，包含构建计划、环境校验、Make/CMake/单文件策略、产物追踪、运行启动器、终端命令与 native/proot 双链路。

- `:core:crash`（`core/crash`）：native 崩溃初始化、崩溃日志展示/上传、自动上传调度、上传状态文案解析和 JobService。

- `:core:debug`（`core/debug`）：调试协议模型和断点存储，定义初始化、继续、单步、回溯、变量、内存、寄存器等调试命令与响应。

- `:core:git`（`core/git`）：Git 服务抽象，包含状态、diff、提交、分支、远端、冲突、进度、凭据和 SSH 管理。

- `:core:linux-distro`（`core/linux-distro`）：Linux 发行版 rootfs 下载、校验、解压、安装、元数据登记、目录探测和 manifest 目录实现。

- `:core:lsp`（`core/lsp`）：LSP 核心能力，包含会话连接、compile_commands 生成/归一化、clangd 配置、远程 LSP、测试场景和诊断模型。

- `:core:ndk`（`core/ndk`）：Android native 工具链与 sysroot 管理，负责资产安装指纹、ABI/API 配置、工具链路径和已安装配置。

- `:core:packages`（`core/packages`）：包管理核心，覆盖内置包安装、包元数据、库路径解析、依赖目录事件、GitHub Registry 包索引读取、APK/下载后端。

- `:core:plugin`（`core/plugin`）：插件系统核心，包含内置插件安装、插件加载/健康检查、诊断、命令、LSP 插件、主题、图标、市场 API 与 Linux 环境桥接。

- `:core:project`（`core/project`）：项目创建与模板安装，维护项目元数据、APK 导出能力判断、内置模板定义和迁移提示。

- `:core:proot`（`core/proot`）：可选 PRoot/Linux 运行时，包含 rootfs bootstrap、健康检查、guest 包管理、安装日志、路径校验与 MobileExec 集成。

- `:core:search`（`core/search`）：代码/JSON/十六进制/项目搜索，提供搜索结果模型、历史记录、批量替换和预览能力。

## Feature 功能模块

- `:feature:ai`（`feature/ai`）：AI 聊天与工具系统，包含渠道配置、API 客户端、流式响应、会话仓储、工具注册/执行、AI 设置桥接和聊天 ViewModel。

- `:feature:editor`（`feature/editor`）：编辑器业务层，管理打开的内容标签、编辑器管理器、文件编码、书签、搜索状态、头文件解析和 C/C++ include 支持。

- `:feature:help`（`feature/help`）：帮助与反馈功能，提供帮助文档仓储、搜索结果、快捷动作、反馈分类、反馈提交和对应 UI 状态。

- `:feature:output`（`feature/output`）：输出面板管理，承接编译输出、日志展示和底部输出区域状态。

- `:feature:packages`（`feature/packages`）：包管理 UI，展示包列表、筛选、安装/卸载/批量安装/更新状态，并调用 `:core:packages` 与 `:core:proot`。

- `:feature:projectlist`（`feature/projectlist`）：项目列表与首页项目管理 UI，包含项目标签、项目操作、更新提示入口状态和项目入口状态。

- `:feature:settings`（`feature/settings`）：设置中心，覆盖编辑器、LSP、编译器、项目、存储、终端、AI、Git、外观、键盘、插件、开源许可证等页面。

- `:feature:terminal`（`feature/terminal`）：终端功能层，管理终端会话、shell 解析、终端偏好、会话快照、zsh/locale/开发包安装和 Termux 视图适配。

- `:feature:tutorial`（`feature/tutorial`）：新手教程与聚光灯引导，包含教程步骤、进度存储、目标注册、聚光灯状态和教程 ViewModel。

- `:feature:viewer`（`feature/viewer`）：文件查看器模型，提供 diff 颜色/并排行、十六进制查看、图片信息和大文本分页状态。

- `:feature:wizard`（`feature/wizard`）：新建项目向导，承接模板选择、项目参数、创建状态和沉浸式向导 Activity。

- `:feature:workspace`（`feature/workspace`）：工作区依赖安装与环境配置，包含工具链配置、依赖安装状态机、安装日志、rootfs 健康状态和安装完成事件。

## 外部本地模块

- `:termux-terminal:terminal-emulator`（`external/termux-terminal/terminal-emulator`）：Termux 终端模拟器核心，提供终端缓冲、ANSI/键盘处理、会话、颜色、宽度计算和 native pty JNI。

- `:termux-terminal:terminal-view`（`external/termux-terminal/terminal-view`）：Termux 终端 View，提供终端渲染、手势缩放、文本选择游标和 View Client 接口。

- `:mobile-exec:runtime`（`external/mobile-exec/runtime`）：MobileExec 运行时 JNI/so 承载模块，主要提供 native 运行资产与 manifest 边界。

- `:mobile-exec:integration`（`external/mobile-exec/integration`）：MobileExec Kotlin 集成层，封装预加载模式、系统 linker 模式与运行时调用入口。

- `:xcrash`（`external/xcrash`）：xCrash 本地源码模块，提供 Java/native/ANR 崩溃捕获、tombstone 管理、解析和回调接口。

- `:immersionbar-local`（`external/immersionbar/immersionbar`）：沉浸式状态栏/导航栏库源码，处理状态栏颜色、键盘适配、刘海屏、导航栏监听和 Fragment 绑定。

- `:immersionbar-ktx-local`（`external/immersionbar/immersionbar-ktx`）：ImmersionBar Kotlin 扩展封装，为 Activity/Fragment 提供更轻量的 Kotlin 调用方式。

- `:devicecompat-local`（`external/devicecompat/library`）：设备兼容信息库，提供品牌、系统、市场名和系统属性读取能力，供权限库等模块使用。

- `:xxpermissions-local`（`external/xxpermissions/library`）：XXPermissions 本地源码，封装 Android 危险权限/特殊权限申请、manifest 检查、权限页面跳转和 Fragment 请求通道。

## Tools 模板模块

- `:tools:template-common`（`tools/template-common`）：模板工程公共能力，主要封装运行时权限分组、权限解析和统一权限申请流程。

- `:tools:template-native-activity`（`tools/template-native-activity`）：Native Activity 模板模块，提供最小 native activity 壳和模板资源。

- `:tools:template-sdl3`（`tools/template-sdl3`）：SDL3 模板模块，提供 SDL Activity 壳与模板资源，用于生成 SDL 图形工程。

- `:tools:template-terminal`（`tools/template-terminal`）：终端模板模块，集成 Termux terminal-view，为模板工程提供终端 Activity 和会话客户端。

## 复合构建与源码包

- `build-logic:convention`（`build-logic/convention`）
  - 功能：主工程 Gradle 约定插件集合。
  - 覆盖：Android library/app 约定、Compose 约定、版本号、ABI 聚合、Guardrails、工具链资产校验、Tree-sitter 查询同步、语言注册表生成、mapping 备份与可选上传。

- `external/mobile-android-tree-sitter`（included build `ATS`）
  - `:android-tree-sitter`：Tree-sitter Java/JNI 绑定，包含 `TSParser`、`TSTree`、`TSNode`、`TSQuery`、`TSLanguage` 等基础封装。
  - `:annotations`：为 native header 生成和同步控制提供注解。
  - `:annotation-processors`：注解处理器，校验 native 方法并生成同步/头文件相关辅助代码。
  - `:app`：ATS 示例/测试应用，不属于 MobileIDE 运行期主业务。
  - `build-logic:ats`：ATS 自身的语法构建插件，负责编译 Tree-sitter grammar、生成 JNI 头、构建调试符号和动态 grammar 模块。
  - 动态 grammar 模块：根工程通过 dependency substitution 使用 `tree-sitter-java`、`tree-sitter-json`、`tree-sitter-c`、`tree-sitter-cpp`、`tree-sitter-bash`、`tree-sitter-yaml`、`tree-sitter-make`、`tree-sitter-cmake`、`tree-sitter-rust`、`tree-sitter-toml`。

## 维护边界

- 新增用户可见功能：优先放到 `feature/*`，共用模型或服务再下沉到 `core/*`。
- 新增底层能力：优先复用 `:core:common`、`:core:model`、`:core:config`、`:core:storage` 等已有抽象，避免重复造轮子。
- 新增 UI 文案：必须走 `:core:i18n` 的字符串资源封装，同步维护默认与英文资源。
- 新增第三方库：需要评估 R8 keep 规则，优先在对应模块 `consumer-rules.pro` 中最小化声明。
- 修改外部本地模块：保持第三方边界清晰，避免把 MobileIDE 业务逻辑写进 `external/*`。
